# Getting Started

Welcome to Autodocs. This guide walks you through generating a local, type-driven API reference and publishing it as a static site.

## 1. Install the CLI

```bash
npm install -D @opensyntaxhq/autodocs
```

## 2. Create a config

```bash
npx autodocs init
```

This creates `autodocs.config.ts` (or `.js`/`.json`) with your include patterns, output directory, and UI theme.

## 3. Generate the docs

```bash
npx autodocs build
```

The build writes a static site into `docs-dist/` with the React UI, `docs.json`, and `config.json`.

## 4. Preview locally

```bash
npx autodocs serve
```

## Customize the UI

Open `autodocs.config.ts` and adjust:

- `theme.logo` and `theme.favicon` to brand the UI.
- `theme.primaryColor` to align with your design system.
- `features.search` or `features.darkMode` to toggle UI capabilities.
- `sidebar` to add custom pages or autogenerated sections.

## Add custom pages

Create Markdown files (for example, `docs/intro.md`) and reference them in `sidebar`:

```ts
sidebar: [
  { title: 'Getting Started', path: '/docs/intro.md' },
  { title: 'API Reference', autogenerate: 'src/' },
];
```

The build step copies those Markdown files into the output so they can be served as regular pages.

## Next steps
